<h2>2. 通信关系（HTML 文字图）</h2>

<pre>
【现有聊天链路（保持不变）】
Web UI ✅
  → HTTP
main_web.py ✅
  → Web API ✅
EventHandler + AgentService ✅
  → LLM ✅
  ↘ WebSessionStore ✅

【实时通话链路（独立入口）】
Realtime UI 🚧
  → HTTP /realtime/sessions
main_realtime.py ✅
  → WS /realtime/ws/{session_id}
RealtimeWsTransport ✅
  → signal.offer / signal.answer / signal.ice
WebRTC PeerConnection(aiortc) ✅（基础协商）
  → 媒体轨（占位视频✅ / 占位音频🚧）
  → webrtc.media.track.ready / webrtc.remote.track.received ✅
  → assistant.text.*（收到远端轨后的确认消息）✅
  → Live2D 事件 🚧
</pre>

<h2>3. 时序（HTML 版）</h2>

<ol>
  <li>Realtime 前端（🚧）调用 <code>POST /realtime/sessions</code>（✅）创建会话。</li>
  <li>服务端返回 <code>session_id</code> 与初始状态（✅）。</li>
  <li>前端连接 <code>WS /realtime/ws/{session_id}</code>（✅），收到 <code>session.joined</code> / <code>call.state.changed</code>（✅）。</li>
  <li>前端发送 <code>signal.offer</code> / <code>signal.ice</code>（✅ 信令通道）。</li>
  <li>信令 transport 转交给 WebRTC 模块（✅），完成 answer/ice 回传（✅）。</li>
  <li>WebRTC 模块输出占位视频轨（✅），并逐步补占位音频轨与 Live2D 驱动事件（🚧）。</li>
</ol>

<h2>4. 当前边界约束</h2>

<ul>
  <li>不改现有主链路（EventHandler/命令分发/聊天流程）。</li>
  <li>不改 QQ 链路。</li>
  <li>仅在 <code>transport</code>、<code>bootstrap</code>、<code>main</code>（及必要实时前端）推进实时能力。</li>
</ul>
